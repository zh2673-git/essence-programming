## 核心公式汇总

### 1. 循环更新公式（核心创新）

```
h_{t+1} = A·h_t + B·e + Transformer(h_t, e)

其中：
- h_t: 第t次循环的隐状态
- e: 来自Prelude的编码输入（每次循环注入）
- A, B: 学习的注入参数矩阵（可训练）
- Transformer: 注意力 + MoE前馈
```

### 2. 稳定性约束

```
谱半径约束：ρ(A) < 1

确保循环不会发散，系统稳定
```

### 3. MLA注意力公式

```
Q = Concat[Q_up(Q_down(x)), Q_rope(x)]
K = Concat[K_up(KV_down(x)), K_rope(x)]
V = V_up(KV_down(x))

其中：
- Q_down/Q_up: Q的低秩压缩/解压
- KV_down/K_up/V_up: KV的低秩压缩/解压
- Q_rope/K_rope: RoPE位置编码
```

### 4. MoE路由公式

```
router_logits = Router(x)
weights, selected = TopK(Softmax(router_logits), k=n_experts_per_tok)

output = Σ(expert_i(x) × weight_i) + Σ(shared_expert_j(x))
```

## 配置参数映射

```
MythosConfig
├── 基础维度
│   ├── vocab_size: 词表大小
│   ├── dim: 模型维度
│   └── n_heads: 注意力头数
│
├── 架构配置
│   ├── max_seq_len: 最大序列长度
│   ├── max_loop_iters: 最大循环次数 ⭐
│   ├── prelude_layers: 前奏层数
│   └── coda_layers: 尾声层数
│
├── MoE配置
│   ├── n_experts: 专家总数
│   ├── n_shared_experts: 共享专家数
│   ├── n_experts_per_tok: 每token激活专家数
│   └── expert_dim: 专家维度
│
├── 注意力配置
│   ├── attn_type: "mla" 或 "gqa"
│   ├── n_kv_heads: GQA的KV头数
│   ├── kv_lora_rank: MLA的KV低秩
│   ├── q_lora_rank: MLA的Q低秩
│   ├── qk_rope_head_dim: RoPE维度
│   ├── qk_nope_head_dim: 非RoPE维度
│   └── v_head_dim: V头维度
│
└── LoRA配置
    └── lora_rank: 低秩适配秩
```

## 设计亮点速览

| 亮点 | 技术实现 | 价值 |
|-----|---------|------|
| 🔁 循环深度 | RecurrentBlock + InputInjection | 参数复用，深度可变 |
| 💉 输入注入 | A·h + B·e 公式 | 防止信息漂移 |
| 🧠 MLA注意力 | 低秩Q/K/V压缩 | 减少KV缓存，加速推理 |
| 🎯 MoE稀疏 | TopK路由 + 共享专家 | 参数高效 |
| 🔄 注意力切换 | MLA/GQA可选 | 灵活适配 |
