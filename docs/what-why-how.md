# 三阶方法论详解 (The Triad Methodology)

## 概述

三阶方法论是本质编程的核心框架，通过**是什么 → 为什么 → 怎么做**的逻辑链条，从第一性原理出发推导软件设计。

---

## 第1阶：是什么 (What) - 定义本质

### 1.1 本质特征（内涵）

从根本问题出发，推导系统本质：

| 推导要素 | 问题 | 目的 |
|---------|------|------|
| **核心问题** | 用户面临什么问题？ | 明确系统要解决的根本问题 |
| **目标用户** | 为谁解决？ | 确定用户画像和使用场景 |
| **核心价值** | 解决后带来什么？ | 量化系统价值 |
| **一句话定义** | 类比理解 | 用简单语言描述系统本质 |

### 1.2 关键属性（外延）

从本质特征**逻辑推导**出系统必须具备的属性：

```
本质特征 → 需要的能力 → 对应的属性（模块）
```

**示例推导**：
- 本质特征 "记忆辅助" → 需要存储任务数据 → 属性：任务模块（Task）
- 本质特征 "状态追踪" → 需要记录谁在操作 → 属性：用户模块（User）

### 1.3 区分属性（边界）

明确**不是什么**，划定系统边界：

| 对比维度 | 本系统 | 对比系统A | 对比系统B |
|---------|-------|----------|----------|
| 核心定位 | ... | ... | ... |
| 功能范围 | ... | ... | ... |

## 第3阶：怎么做 (How) - 实现交付

### 3.1 模块划分（从属性推导）

```
项目
├── 模块A
│   ├── 属性：...
│   └── 方法：...
├── 模块B
│   ├── 属性：...
│   └── 方法：...
```

### 3.2 方法设计（对属性的编排）

```python
def method_name(param: Type) -> Result:
    """方法说明
    
    利用的属性：
    - PropertyA：用途说明
    - PropertyB：用途说明
    
    逻辑流程：
    1. 步骤一
    2. 步骤二
    3. 步骤三
    """
```

### 3.3 路由设计（方法的多场景分发）

同一场景，不同策略：

```python
class Router:
    def dispatch(self, action: str, **kwargs):
        handlers = {
            "action_a": self._handle_a,
            "action_b": self._handle_b,
        }
        handler = handlers.get(action)
        return handler(**kwargs)
```

### 3.4 接口设计（跨模块契约）

```python
from typing import Protocol

class ServiceInterface(Protocol):
    """服务接口契约"""
    
    def method(self, param: Type) -> Result:
        """方法契约说明
        
        Args:
            param: 参数说明
            
        Returns:
            返回值说明
            
        Raises:
            ExceptionType: 异常说明
        """
        ...
```

## 递归执行

### 递归逻辑

当属性标记为"需深入"时，对该属性递归执行三阶分析。

### 判断标准

| 标记 | 条件 | 后续动作 |
|-----|------|---------|
| **需深入** | 复杂度高/独立性强/复用性高 | 生成子文档，递归执行三阶分析 |
| **见下方** | 简单属性 | 当前文档详细展开 |

### 停止条件

- 递归深度达到3层
- 当前层级没有"需深入"的属性
- 属性已经是基础类型

---

## 核心原则

1. **逻辑链条**：每一阶的推导必须基于上一阶的结论
2. **本质优先**：从根本出发，杜绝随意性
3. **属性驱动**：模块从属性推导，方法从属性编排
4. **契约协作**：接口定义跨模块协作契约
